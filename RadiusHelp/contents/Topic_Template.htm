Sizing and Positioning

The following information describes how to position and size your elements.

Sizing

You should  avoid using pixels to size images, this causes them to be static so that they will not respond to screens with different dimensions or resolutions. 

Instead, you should choose from the following units, to make sure that the topic displays properly in different environments:

  • %: Sizes the image based on the parent container. For example, if the image has no parent element, a 100% width will make it as wie ad the topic page. If the image is in a container that is 50% of the topic page width, then givng it a 100% size will make it 50% of the topic page width. 
  • em: Sizes the image based on the font-size of the parent element. 1em is equal to the current font-size of the element in question. If you haven’t set font size anywhere on the page, then it would be the browser default, which is probably 16px.  So by default 1em = 16px.
  • rem:  While em is relative to the font-size of its direct or nearest parent, rem units determine the pixel size depending on the font size of the root element of the page, i.e. the html element. That root font size is multiplied by whatever number you’re using with your rem unit. For example, with a root element font size of 16px, 10rem would equate to 160px, i.e. 10 x 16 = 160.
