Page dividers are a visually pleasing way to separate your topic content. I’ve created styles for a primary divider used to separate unrelated chunks of content, and a minor divider to create breaks in content that is related.
RoboHelp has a button which adds a horizontal line, so you just need toa dd the class name to that in the Attributes section of the right-hand panel.

Adding Page Dividers

The following instructions describe how to add page dividers to your topic:

  1. Select the line in a topic where you want to add a page divider.
  2. Click Insert HTML Element   on the toolbar then select Horizonal line. A hr element is added to the page.
  3. Select the hr element and do one of the following depending on which divider you want to add:
  • To add a primary page divider, enter hrule in the Class field in the Attributes section of the right-hand panel.
  • To add a secondary page divider, enter hrule-inner in the Class field in the Attributes section of the right-hand panel.
  1. Tab out of the field to update the hr style.

Sizing and Positioning

The following information describes how to position and size your elements.

Sizing

You should  avoid using pixels to size images, this causes them to be static so that they will not respond to screens with different dimensions or resolutions. 

Instead, you should choose from the following units, to make sure that the topic displays properly in different environments:

  • %: Sizes the image based on the parent container. For example, if the image has no parent element, a 100% width will make it as wie ad the topic page. If the image is in a container that is 50% of the topic page width, then givng it a 100% size will make it 50% of the topic page width. 
  • em: Sizes the image based on the font-size of the parent element. 1em is equal to the current font-size of the element in question. If you haven’t set font size anywhere on the page, then it would be the browser default, which is probably 16px.  So by default 1em = 16px.
  • rem:  While em is relative to the font-size of its direct or nearest parent, rem units determine the pixel size depending on the font size of the root element of the page, i.e. the html element. That root font size is multiplied by whatever number you’re using with your rem unit. For example, with a root element font size of 16px, 10rem would equate to 160px, i.e. 10 x 16 = 160.

Positioning

To properly position elements in your topic, you need to get comfortable the CSS box model, but most notably margins and padding. A margin is the space between the edge of your element and its parent, while padding is the space between the content of your element and its edge. 

For convenience, you can add the class "center-horizontal" to an element using the Class field in the Attributes section of the right-hand panel to horizontally center it in your topic.

Note: This style has a display value of "block" so that multiple elements with this class with stack on top and won't display inline. To center multiple elements you will need to add a container element around them and then style them relative to that container.
