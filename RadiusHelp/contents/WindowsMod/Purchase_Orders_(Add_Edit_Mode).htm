Purchase Orders (Add/Edit Mode)

Use the Purchase Orders (Add/Edit Mode) page to amend existing purchase orders which can then be sent to suppliers and add or amend purchase order lines. The purchase order price is used by invoice checking to verify that the supplier’s purchase invoice is correctly priced.

Purchase Order Details

The Purchase Order Details section contains the main details of the purchase order record; holding the order number, order date, the required date, order status and other details.  

Fields (Purchase Order Details)

Field

Description

Purchase order Number

This is the unique number which identifies the purchase order in the EFI Radius database. This will be automatically assigned when you save the new purchase order on this tab.

Note: Auto-assigning purchase order numbers can be disabled by selecting Company > Options > Company Level Options from the navigation pane, and specifying Manual in the Automatic purchase order numbering field of the Purchasing and Outsource tab.

Version

    
Displays an incremented version number if the purchase order is changed after it has been printed, then allows you to enter text to explain the reasons for the change. The field next to the version number will be automatically updated by EFI Radius and it indicates the print status of the purchase order.

Print Status

 This is automatically updated by EFI Radius and it indicates the print status of the purchase order.

Supplier Code

Specify the supplier account code.

If you are using preferred suppliers, defined by clicking Item Preferred Suppliers  on the Items Details tab of the Items > Items window, the supplier may be automatically populated from the item record when you create the first purchase order line. If you want it to be automatically populated, leave this field blank.

Note:  When the supplier is selected, most of the remaining fields will fields will automatically default from the supplier record, but can be amended if required.

Total PO Value

    Displays the total purchase order value, given as the sum of the PO lines on the Order Lines tab.

Status

Select one of the following statuses:

  • On Hold: the purchase order cannot be sent to the supplier because it is incomplete or not authorized.
  • Authorized: The purchase order can be sent to the supplier.
  • Confirmed: The purchase prices have been confirmed and cannot be changed. EFI Radius will automatically change the purchase order status to Confirmed when the first shipment from the supplier is received so that the inventory values cannot be changed.
  • Complete: The purchase order is complete.
  • Cancelled: The purchase order has been cancelled.

Note: If In use is selected from the PO Approvals in use field on the Purchasing and Outsource tab of the Company > Options > Company Level Options window, you will not be able to select a status. One of the following status options will be automatically selected depending on whether your PO has been sent for approval:

  • Not sent for approval: The purchase order has been created but not sent to a third-party for approval.
  • Pending approval: The purchase order has been sent to a third-party for approval but not yet been approved.

Currency

Displays the currency that will be used for the purchase order prices. This will default from the selected supplier and cannot be changed.
Supplier Reference

Enter any references that the supplier may have given you, such as a quotation reference.

Your Contact

Specify the contact at the supplier's address.

Our Contact

Specify the contact at your company. This defaults to your user name but it can be changed if necessary.

Language

Select the language for printing the purchase order document and labels. This defaults to the language of the supplier.

Delivery Terms

    
Select the language for printing the purchase order document and labels. This default selects the description of the method of delivery from this field. Depending on your business practices, this can be used in the following ways:

  • Used to indicate the freight cost to the supplier for insurance purposes.  
  • Used for Intrastat purposes where the European Union requires that you tracks movement of goods for tax purposes.

This defaults from the supplier record (Company > Customers and Suppliers > Suppliers) but may be changed if required. You can set up and amend delivery terms in the System > General Set-up > Terms of delivery window. to the language of the supplier.

Terms Code

Specify the settlement terms that the supplier allows. This defaults from the supplier but can be changed.

Order Date

Specify the date that the order is being placed on. For new and copied purchase orders, this defaults to today's date.

For intercompany transfers, this date defaults to the order date from the sales order.

Date Required

Specify the date by which the order is required.

For intercompany transfers, this date defaults to the required date from the sales order.

Note:  Individual purchase order lines on the Lines tab also have a required date. When you now add a purchase order line on the Lines tab, the Date Required field from the Header tab automatically populates the Date Required field on the Lines tab. Also, if you change the date on the Header tab and then click Save, a prompt will display asking if you want to update all purchase order lines on this purchase order to the updated date.

Country of Origin

Specify country the purchase order was raised in.

Last Date Change

Optionally, specify the date at which the supplier will not allow any amendments to the purchase order. A warning message will be displayed if the purchase order is amended after this date.

Price Group

Use this field to combine purchase order lines together for group pricing. For example, if the price per ton for a material changes when you purchase a greater tonnage it might be possible to get a better price by combining together the quantities of different sizes of the same material. EFI Radius will automatically create the first price group; therefore the initial value of the first purchase order line must be left at zero. For subsequent purchase order lines the price group number will be incremented but it can be changed.

Example:

Purchase Price Table for Material AAA

  • Price per ton for less than 1 ton = 450.00
  • Price per ton for 1 ton and above = 350.00

Purchase Order Line 1 for material AAA 12” wide

  • 0.871 tons – Price Group 1 Price = 450.00

Purchase Order Line 2 for material AAA 10” wide

  • 0.234 tons – Price Group 2 Price = 450.00

If the Price Group for Line 2 is changed to Price Group 1

1.105 tons – Price Group 1 Price = 350.00

Price Group Tab

Use the Price Groups tab to:

  • Group several order lines for the same Item together so that the cumulative quantity is used for the purchase price.
  • Override the purchase price table or enter a price when there is no price schedule.
  • Record the suppliers confirmation of the order.

When you first enter a purchase order line, it has a price group of zero. When you save the line, it creates a price group record. If you enter a second line for the same item you can link it to the same price group. The system will then use the cumulative order quantity for the item when determining the purchase price from the price schedule.

Price Groups Section

The Price Groups section displays a list of your available price groups. Selct a price group from the list to display its information in the Price Group Details section.

The Version Comments tab allows you to attach internal and external comments to each version of the purhase order. External comments print on the purchase order, internal comments do not.
