Purchase Orders (Add/Edit Mode)

Usde the Purchase Orders (Add/Edit Mode) page to amend existing purchase orders which can then be sent to suppliers and add or amend purchase order lines. The purchase order price is used by invoice checking to verify that the supplier’s purchase invoice is correctly priced.

Purchase Order Details

The Purchase Order Details section contains the main details of the purchase order record; holding the order number, order date, the required date, order status and other details.  

Fields (Purchase Order Details)

Field

Description

Purchase order Number

This is the unique number which identifies the purchase order in the EFI Radius database. This will be automatically assigned when you save the new purchase order on this tab.

Note: Auto-assigning purchase order numbers can be disabled by selecting Company > Options > Company Level Options from the navigation pane, and specifying Manual in the Automatic purchase order numbering field of the Purchasing and Outsource tab.

Version

    
Displays an incremented version number if the purchase order is changed after it has been printed, then allows you to enter text to explain the reasons for the change. The field next to the version number will be automatically updated by EFI Radius and it indicates the print status of the purchase order.

Print Status

 This is automatically updated by EFI Radius and it indicates the print status of the purchase order.

Supplier Code

Specify the supplier account code.

If you are using preferred suppliers, defined by clicking Item Preferred Suppliers  on the Items Details tab of the Items > Items window, the supplier may be automatically populated from the item record when you create the first purchase order line. If you want it to be automatically populated, leave this field blank.

Note:  When the supplier is selected, most of the remaining fields will fields will automatically default from the supplier record, but can be amended if required.

Language

    
Select the language for printing the purchase order document and labels. This defaults to the language of the supplier.

Delivery Terms

    
Select the language for printing the purchase order document and labels. This defaultsSelect the description of the method of delivery from this field. Depending on your business practices, this can be used in the following ways:

  • Used to indicate the freight cost to the supplier for insurance purposes.  
  • Used for Intrastat purposes where the European Union requires that you tracks movement of goods for tax purposes.

This defaults from the supplier record (Company > Customers and Suppliers > Suppliers) but may be changed if required. You can set up and amend delivery terms in the System > General Set-up > Terms of delivery window. to the language of the supplier.

Terms Code

Specify the settlement terms that the supplier allows. This defaults from the supplier but can be changed.

Order Date

Specify the date that the order is being placed on. For new and copied purchase orders, this defaults to today's date.

For intercompany transfers, this date defaults to the order date from the sales order.

Date Required

Specify the date by which the order is required.

For intercompany transfers, this date defaults to the required date from the sales order.

Note:  Individual purchase order lines on the Lines tab also have a required date. When you now add a purchase order line on the Lines tab, the Date Required field from the Header tab automatically populates the Date Required field on the Lines tab. Also, if you change the date on the Header tab and then click Save, a prompt will display asking if you want to update all purchase order lines on this purchase order to the updated date.

Country of Origin

Specify country the purchase order was raised in.

Additional Fields (Purchase Order Details)

Field

Description
Supplier Reference

Enter any references that the supplier may have given you, such as a quotation reference.
Last Date Change

Specify the date at which the supplier will not allow any amendments to the purchase order. A warning message will be displayed if the purchase order is amended after this date.

Comments and Addresses (Purchase Order Details)

Field

Description

Purchase order Comments

Enter any comments that are required to print on the purchase order. 

In the Purchase Order Comments window, accessed by clicking Add from template, you can enter text or add standard predefined phrases. 

Note: For intercompany transfers, this field populates with the source company sales order comments.

Delivery Specifications

Enter location specific delivery comments attached to this purchase order. 

In the Purchase Order Comments window, accessed by clicking Add from template, you can enter text or add standard predefined phrases. 

Approval Comments

 This is automatically updated by EFI Radius and it indicates the print status of the purchase order.
Supplier Address

Specify the address of the supplier of the purchased goods.

 Annotations (Purchase Order Details)

 

Order Details

The Order Details section contains the lines for the items on the sales order. If the sales order is linked to an estimate, the lines will be automatically created from the estimate details.
