Advanced Search Filters

Use the Advanced Search Filters window to search for records using queries sent to the Elastic Search engine, allowing you to build and perform search queries that return results incredibly fast, even when searching large databases.

To display this window,  click  next to the search bar at the top of the Radius window. 

Note:  For more detailed information, see About the Advanced Search and How to Use the Advanced Search . 

When your changes are complete, click Done to close this window.

Field

Select a field to search against.

Type

Select the strictness of the search. Choose from:

  1. Must: The results must equal the specified criteria. See the examples listed above.

  2. Must not: The results must not equal the specified criteria. For example, to exclude Flexible estimates from your search results, use the following criteria.

  3. Should: This prioritizes the results of the search results that have been returned with optional criteria. For example, if you have results returned that have a range of 2.000 to 4.000, you can prioritize 3.785 to the top of the results list.

Operator

Select which search to perform against the Value. Choose from:

  1. Range: The value must be between the specified limits.

  2. Match: The results have to contain this word.

  3. Exists: The estimate must have the specified field populated.

  4. Prefix: The word in the specified field must begin with this. If there are two words in the field then either word can begin with this.

  5. Fuzzy: The word in the specified field must be similar to this. This is useful for spelling mistakes.

  6. Wildcard: You can search using wildcards, meaning that the a symbol can be used to replace or represent one or more characters. The most common wildcards are the asterisk (*), which represents one or more characters and question mark (?) that represents a single character.
