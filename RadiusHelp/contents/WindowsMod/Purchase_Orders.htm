Purchase Orders

Use the Purchase Orders page to create new purchase orders or amend existing purchase orders which can then be sent to suppliers. The purchase order price is used by invoice checking to verify that the supplier’s purchase invoice is correctly priced.

The Purchase Orders page is composed of workspace and grid sections for both purchase orders and purchase requirements, which are accessed by hovering over Purchase Orders  on the navigation pane, then selecting one of the following options to open the respective section:

  • Workspace - Purchase Orders: Click to open the My Workspace (PO) Section. 
  • Workspace - Purchase Requirements:  Click to open the My Workspace (PR) Section.
  • Purchase Orders: Click to open the Purchase Orders (grid) section.
  • Purchase Requirements: Click to open the Purchase Requirements (grid)section.
  • ASN: Click to open the ASN (advanced shipping notes) section. 

My Workspace (PR) Tab

The purchase requirements workspace provides a convenient location to quickly filter and create your purchase requirements. The page is divided into filtered columns, which display purchase requirements based on the search criteria of that filter. 

You can drag and drop purchase requirements into the Add to PO Line column, then select one of the following to generate an order for the requirement:

  • Generate PO > Create New PO: Opens the Generate PO window to create a new purchase order for that requirement.
  • Generate PO > Add to Exisitng PO: Opens the Generate PO - Add to Existing window to add the requirement to an existing purchase order.

My Workspace (PO) Tab

The Purchase Orders workspace provides a convenient hub in which to view and access your purchase order records. The page is divided into filtered columns, which display purchase requirements based on the search criteria of that filter. 

By default, the following filters are provided and are based on the status of the purchase order:

  • On Hold
  • Authorized
  • Confirmed
  • Cancelled

Purchase Orders grid Tab

Use the Purchase Orders browser page to search for purchase order records. You can click a Purchase Order Number to open that record or click Create Purchase Order to create a new purchase order record. 

 

Purchase Requirements grid Tab

Use the Purchase Requirements browser to search for purchase requirement records. You can click a Requirement ID to open that record. 

You can select the check box next to a range of requirements, then click one of the following to generate an order for the requirement:

  • Create New PO: Opens the Generate PO window to create a new purchase order for that requirement.
  • Add to Exisitng PO: Opens the Generate PO - Add to Existing window to add the requirement to an existing purchase order.

Advanced Shipping Notes Tab

The Advanced Shippign Notes (ASN) tab contains all your advanced shipping notes and lines, and allows you to create new or edit existing ones. ASNs contain information received from a supplier prior to a delivery advising you of the details of the goods that have or are in the process of being shipped to you. You can enter the details of each line of the shipment before the delivery arrives. This will allow prior preparation of documentation, such as barcode labels etc, to expedite the goods inwards procedure.

Notes Section

The Notes section displays your created ASNs. You can use the search bar to filter by ASN number and click + Create to create a new note. 

ASN Fields Section

Use the ASN fields section to enter infomration for new ASNs or updating information about existing ones.

Field

Description

Advanced Shipping Note Number

The unique number that identifies this ASN. This number is automatically assigned.

Linked to purchase order (check box)

Select to link the ASN record to a purchase order. You can link each line of the ASN to a purchase order line on the ASN Details tab.

Status

This indicates whether goods had been received for an ASN by having one of the following statuses:

  • No Lines: This status is given to an ASN record when it is first created and has no lines.  
  • Outstanding: Indicates that you have not received any goods or you have only received a portion of the goods for the ASN.
  • Completed: Indicates that you have received the goods for all lines on the ASN record.  

Note:  If an ASN line is received and some or all of the other lines for that ASN have not been received then the status will remain as Outstanding.

Supplier Code

Specify the supplier that the goods are coming from.

Delivery Reference

 Enter the supplier’s delivery reference.

Transport ID

 Enter the delivery vehicle's identification, such as a license plate number.

Redirect ASN (check box)

Select to specify another plant as the delivery destination for the ASN. When selected the New plant code field is enabled.

New Plant Code

Specify a different plant as the delivery destination for the ASN. This field is only visible when the Redirect ASN check box is selected.

Note Details Section

The Note Details section consists of a grid which displays each line on the selected ASN. You can either click Add to create a new ASN line or double-click a line in the grid an exisitng one to edit it; both of which display the ASN line fields.

Use the filters to filter the data displayed in the grid.

This section displays a grid containing purchase requirements that match the criteria of the option you slected in the Filter section. The free stock planning grid is displayed by default on this tab. However you can switch to column view by clicking the Column view button. 

Free Stock Planning Colums Section

This section displays the purchase requirements  of every filter in the Filter section, sparated into columns. The free stock planning grid is displayed by default on this tab, however you can switch to column view by clicking the Column view button. 

Inventory Forecast Tab The Inventory Forecast tab displays a breakdown of your inventory supply and demand while providing recommendations for how much inventory to purchase to satisfy future demand.
