Sales Orders

Use the Sales Orders > Sales Orders window to record customer’s purchase order details, agreed selling prices, extra charges and shipping details.

Sales Order details

Buttons

Button

 Description
 Sales Order Analysis

Click to open the Sales Order Analysis window to maintain sales order analysis codes.  These codes are used for transaction analysis.
User Defined Elements

Click to open the User Defined Elements where you can view user defined elements (custom fields) associated with the sales order. User Defined Elements are additional fields which you can define and add into the EFI Radius database. The purpose of the fields is to allow you to record additional information which can be viewed or added to customized reports. See Company > Jobs and Costing > User Defined Elements.
View Quantities by Plant

Click to open the Quantities By Plant window to view which plants the order has been allocated to.
Interface Order Details

Click to open the Interface Order Details window where you can maintain interface details when sales orders are created from a third party system.

Sales Order Notes

Click to open the Sales Order Notes window

Files Attachment

 Click to open the File and Link Attachments

Intercompany Transaction Details

Click to open the Intercompany Transaction Details window

Have Ready Date

Specify the date by which the order needs to be available for shipment in order to meet the delivery date. The default calculation is the Required Date less the prior days. 

Yes

Estimate Code

If a production estimate exists, specify the production estimate number. The sales order details and lines will be created from the estimate. 

Note: If you use the EFI Radius - EFI iQuote integration, this estimate code will likely refer to an estimate created in EFI iQuote. This is only possible when an iQuote integration has been set up, and made active in the Integration Options window.

No

Version

Select which version of the estimate is to be used to create the sales order. 

No

Job Code

The job linked to this sales order. This field is populated when the job is generated automatically with the sales order or when a job is created at a later point and linked to the sales order.

No

Document Language

Select the language to print reports and documents in when you print them from the Sales Orders window. 

 No

Sales Order

The unique number to identify this sales order. This number is automatically generated when the sales order is saved. 

 No

Your Contact

Specify the contact at the customer. 

 No

Order Status

The current status of the order. This defaults to Authorized but can be set to Held, or Credit held. Complete would not be appropriate for a new order entry.

 No

Job Status Code

The status of the production job. The field will normally be automatically populated based on the linked job. 

 No

CSR

The customer service respresentative code. This will default to the CSR code maintained on the Customer Representative table. 

 No

Customer Order Date

Enter the date that the customer placed the order. This will default to today's date but can be changed as needed.  

 No

Entered By

The name of the Radius user who entered the sales order. This will default to the user that is currently logged in. 

 No

Sub-Client Name

Enter the name of the sub-division of the customer that this order is for. 

 No
