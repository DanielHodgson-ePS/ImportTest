Sales Order Version

The first version of the sales order is zero. Once the order acknowledgement is printed, any changes made to the sales order increment the version number. Notes relating to the change can be entered using the Version Comments button at the bottom of the page. The order acknowledgement status is displayed alongside the version number. These are:

  1. Not printed

  2. Printed

  3. Amended not printed 

 

Payment Mode

This field displays the status of credit card orders:

  1. Payment Request Sent  - Indicates that the billing process has been run and a payment request has been sent to the integrated product.

  2. Payment Rejected- Indicates that integrated product cannot process the credit card payment. You should contact your customer, and diagnose the issue or arrange payment via a different method.

  3. Payment Received - Indicates that the payment has been sent to your account from your customer's credit card.

  4. Authorization Request Sent - Indicates that your customer's credit card details have been entered into the Wells Fargo Payment Gateway.

  5. Authorization Accepted - Indicates that Wells Fargo has sent confirmation that there is sufficient credit for the payment.

  6. Authorization Declined - Indicates that Wells Fargo has found that the credit is insufficient for the payment.You should contact your customer, and diagnose the issue or arrange payment via a different method.

For more information on the Wells Fargo Payment Gateway Interface, contact EFI support. 
 

 

Rep

Sales representative code; this will default to the sales representative code maintained on the Customer Representative table.

 

Day Prior

The number of days the order is to be ready before the customer's date. This will default on selection of the ship to address if lead time has been defined for the address. You can exclude non shipment days from the lead days calculation by using the Non ShipmentDates/Days window.

 

EDI Change Status

 

 

Currency

The currency in which the sales order is priced. This will default from the customer record.

If the Status field under GL Interface is set to XLink Only on the General Ledger tab of the Company > Options > Company Level Options window, you can change the currency to be different to the currency of the customer. After saving the sales order, you can update the sales order currency until an order line item is created for this order. Then, you will no longer be able to change the currency.  

When the currency of the customer and sales order is different, the currency will be automatically converted to the currency of the customer when displayed in the Sales order balance field on the Detail tab of the Company > Customers and Suppliers > Customers window. 

Note:  If you have the iQuote integration enabled in the Integrations > Integration Options window, changing the currency is not supported in that workflow even if the Status under GL Interface is set to XLink Only on the General Ledger tab of the Company > Options > Company Level Options window.

Columns / Fields

Field / Column

Description

Required

Item Code

Specify the item required by the customer. All sales order lines must be a valid item code. 

Yes

Quantity

 

Yes

Price

 

Yes

Required Date

Specify the date when the customer requires the order. This date will default from the Required Date in the main sales order details.

Yes

Have Ready Date

Specify the date when the finished products must be available for shipment. This date will default from the Have Ready Date in the main sales order details. 

Yes

Size

Enter the dimensions of the roll or sheet in millimeters or inches.

These fields will only be active for sheet  (Class 2 - Sheet material) or roll (Class 1 - Roll material) products. This is normally only relevant for raw materials sold to the customer. 

When you specify a roll or sheet material in the Item code field that has a default size to an order line, the default size will automatically populate in this field. This will be from the size defined as the default size in the Items > Set-up > Standard Substrate Sizes window.

 

Item Status

The current status of the inventory for this sales order line. The status will normally change from Unplanned, to Planned when the sales order line has been reserved against a production job, then to In Inventory when sufficient inventory is available, and finally to a status of Complete.

 

Consignment Stock (check box)

Select to indicate that the item is to be held as consignment inventory at the customer’s premises. The item’s inventory will be shipped to the customer but not billed until used by the customer. This option will be set automatically according to the settings on the consignment flag of the item but is still available for updating.

 

Customer Item Reference

Optionally, enter the customer’s reference for the item that they require or click  to select from a list of finished productitems which are linked to the customer. Items are linked to customers using the Customer code field on the Produced/Sold Details tab of the Items window.

 

Customer PO Line

Optionally, enter the customer's PO line reference. This is sometimes required by some customers as a means of identifying the purchase order line in their computer system. 

 

Run on Price

If required, enter a selling price per unit of measure (usually per thousand) for additional quantities above the ordered quantity. The run on price is based on the variable cost of the additional quantity. The setup costs having already been recovered within the order quantity.

The unit of measure is displayed to the right of the Run On Price field. 

 

Ordered Value

This field is calculated by EFI Radius from the Quantity and Price fields. To override the value, select the Lot Priced check box. 

 

Proof Required Date

Optionally, specify the date when approved proofs are required from the customer. This is for information only and has no operational function in EFI Radius.

 

Order Type

Optionally, select whether the sales order line is for a new item or repeat work. 

 

EDI Change Code

 

 

Production Grouping Number

A production grouping number is assigned to a grouping of production requirements by pricing combination. When sales orders are priced, the price is often influenced by the fact that certain items may be run in combination (minimizing make ready costs). 

 

Unders %

Optionally, enter the under production percentage which is acceptable to the customer.  This will default from the Unders percent field on the Details tab of the Company > Customers and Suppliers > Customers window.

 

Overs %

Optionally, enter the over production percentage which is acceptable to the customer. This will default from the Overs percent field on the Details tab of the Company > Customers and Suppliers > Customers window. 
 

 

End Use

 

 

Lot Priced (check box)

Select to indicate that the sales order line will be sold for a fixed value no matter what quantity is eventually delivered. Selecting this option activates the Ordered Value field and deactivates the Price field.
