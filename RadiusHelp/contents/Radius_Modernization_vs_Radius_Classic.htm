Radius Modernized vs Radius Classic - Enhancements and Differences

The release of the Radius Modernization project provides wide selection of new features and enhancements, as well as introducing quality of life workflow changes.

The following information provides an in depth comparison between the modernized and classic Radius releases, describing the changes that have been implemented and provides high-level instructions on using the new functionality.

Modernized User Experience and User Interface Design

The most fundamental change to the modernized Radius release is the sleek and intuitive redesign of the application front-end. Not only does this provide a more attractive and easy to use work environment, but also gives you complete control over the which fields are available in your Radius windows and how they are displayed, in the form of "views".

 

Navigation

The modernized interface means that navigating Radius has never been easier, it's now much quicker to find the pages and information you need. The following new and enhanced features have been introduced to facilitate this:

  1. Many of the fields (such as sales order numbers) are hyperlinked, and open their respective page when clicked. 
  2. An advanced version of the Radius has been introduced (modeled after the Radius Classic Estimating  Advanced Search), which allows you to build fast and powerful search queries.
  3. Some of the Radius classic windows have been re-imagined as tabs, so that you can more easily follow your workflow without additional navigation. For example, the Radius classic shipping and invoice functionality have been encapsulated by the Shipping and Invoice Details tabs on the Sales Order page.
  4. You can now declutter your page view with the introduction of collapsible details - simply click Hide  next to a section to collapse it, then click Show  to expand it when you want to see the details. 

Advanced search

The search bar at the top of the page provides the standard Radius navigation search, however you now can build advanced search queries (like you can in the Estimating Advanced Search) for your other records. 

To access the Advanced Search Filters window, click Advanced Search  next to the Search Sales Order lookup.
 

Note     For detailed instructions and information on the advanced search, see Navigation.

Sales Orders View Customization

Views allow you to tailor make Radius windows and can be quickly and easily changed on the fly. Additionally, administrators can control which views are available to users, and set a preferred view as default.

 

In the classic Radius system, you had to manually search for sales order records and filter them by their Status in order to view a refined list of them. You then had to slelect each record you wanted to edit and click multiple buttons in order to edit them.

In the Modern Radius system, you can simply click one of your predefined views in order to filter which sales order records are displayed. You can then quickly and easily inline edit each record.

Shipping Differences

While the workflow for shipping remains primarily the same, there have been some key changes made to bring continuity to how shipping requests are generated, which also provides a far more dynamic approach to how to perform shipments.

Header – Lines Relationship

In the classic version of Radius, while shipping requests generated in the Sales Orders > Shipping Requests window followed a header – lines relationship, requests generated by clicking the Generate Shipping Requests button on the Order Lines tab of the Sales Orders window did not.

In the modernized version, only the header – shipping lines method of shipping request generation is used. Furthermore, you can attach shipping lines to a header that were generates from different sales orders and plants; the header is simply details of the shipment; the lines are anything you want to send with that shipment.  

Additionally, whereas in the old Radius system you had to enter the amount of goods to ship as a percentage of the requirement, you now just need to enter the number of goods you are shipping.

Address creation limitation

In the classic Radius system you could create a new address when adding one to a sales order, by clicking the Add/Edit button in the Address Lookup window, accessed from the Order Details tab of the Sales Orders > Sales Orders window. 

However this feature is not yet implemented in the modernized version of the Radius application.

Switching Between Metric / Imperial 

Switching between metric and imperial measurements in the classic Radius system was quite inconvenient. To switch the unit, you had to select or deselect the Metric check box on the General tab of the Company > Options > Company Level Options window, which updated every size field for the entire currently signed in company. 

Now, you can easily swap between imperial and metric values by simply clicking the Imperial  and Metric  buttons next to any Size field.
