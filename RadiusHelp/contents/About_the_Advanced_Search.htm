Radius Tutorials - About the Advanced Search

The Radius Advanced Search utilizes the Elastic Search engine, allowing you to build and perform search queries that return results incredibly fast, even when searching large databases. You can access this window by clicking Advanced Search  next to the search field on the toolbar.

About Query Building

Search queries are built against a Field, for which you select a Type, Operator, and enter or select a value. Multiple criteria can be added for a query, allowing you to create complex searches. Each criteria you define is automatically saved, so it can be easily used again, and can be edited or deleted from within the Advanced Search Filters window.

Operators

An operator in the Advanced Search is the element that tells the search to perform a specific relational or logical operation and produce final result. Simply put, an operator is used to define the conditions of the search you want to perform. The following information gives a detailed breakdown and example for each of the operators available in the Advanced Search.

Match

The results have to contain this word.

The following example is looking for an estimate that has a Product group that contains the word Boxes. Results for this includes Boxes or Food Boxes

Field

Type

 Operator Value From Value To Value
Producr group Must Match Boxes

Range

The value must be between the specified limits.

The following example is looking for an item with a Dim A that is greater than 4.000 but less than or equal to 12.000. 

Field

Type

 Operator Value From Value To Value
Dim A Must Range
4.00 12.00

Wildcard

You can search using wildcards, meaning that a symbol can be used to replace or represent one or more characters. The most common wildcards are the asterisk (*), which represents one or more characters and question mark (?) that represents a single character.

The following example is looking for an estimate that has an Estimate Title that contains the exact word coating using an operator of Match.

Field

Type

 Operator Value From Value To Value
Estimate Title Must Match Coating

The results show the following estimate titles:

  • Coating and Slitting
  • Extrusion Coating with Drool

However, if you use an operator of Wildcard and look for an estimate that has an Estimate Title that contains coating*, the asterisk * can represent multiple characters.

Field

Type

 Operator Value From Value To Value
Estimate Title Must Wildcard Coating*

The results show the following estimate titles:

  • Coating and Slitting
  • Coatings Multi pass
  • Carton Coatings
  • Extrusion Coating with Drool

Exists

The estimate must have the specified field populated.

The following example is looking for an estimate that has a Customer code specified.

Field

Type

 Operator Value From Value To Value
Cusotmer Code Must Exists


Prefix

The word in the specified field must begin with this. If there are two words in the field then either word can begin with this.

The following example is looking for an estimate that has a Product group that contains a word staring with Pack. For example, if you search for Pack then the results could include Packaging or Flexible Packaging.

Field

Type

 Operator Value From Value To Value
Product Code Must Prefix Pack

Types

Types define how you want to match your data, essentially determining whether it must or should include, or whether it should exclude, the results you are querying for. The following information gives a detailed explanation and example of the search types available in the advanced search. 

Must

The results must equal the specified criteria. See the examples listed above.

Must Not 

The results must not equal the specified criteria. For example, to exclude Flexible estimates from your search results, use the following criteria.

Field

Type

 Operator Value From Value To Value
Product Group  Must not Match Flexible    

Should

This prioritizes the results of the search results that have been returned with optional criteria. For example, if you have results returned that have a range of 2.000 to 4.000, you can prioritize 3.785 to the top of the results list.

Field

Type

 Operator Value From Value To Value
Dim A Must  Range   2.000 4.000
Dim A  Should Match 3.785
